# Source Grounding（来源锚定 / 防知识幻觉）

简体中文

这个目录下的 Prompt 旨在抑制大模型在引用事实性信息时的**知识幻觉（Knowledge Hallucination）**——即编造不存在的论文、API、版本号或项目名称。

## 📖 产生背景与作用

事实性幻觉是 LLM 最臭名昭著的结构性缺陷之一。与本仓库中其他 Prompt 所针对的缺陷不同，它既不是逻辑推理层面的错误（`logic-correction` 的靶向），也不是新理论验证层面的妄想（`anti-delusion` 的靶向），而是一种更底层的**"引用层幻觉"**——模型在陈述"已有事实"时，以极高的确信度编造一个从未存在过的东西。

这种幻觉的机制根源在于：**LLM 是基于统计模式生成文本的，不是在查询一个事实数据库。** 当模型在训练中见过大量"论文引用"的文本模式，它就学会了生成看起来像论文引用的文本——完整的标题、作者、甚至 DOI，但这些可能根本不存在。对模型来说，"看起来像正确答案"和"真正正确的答案"在生成概率上没有本质区别。

这在工程和研究场景中造成的危害是直接且严重的：
- **幽灵依赖**：模型推荐了一个不存在的 npm 包或 Python 库，你花 30 分钟排查为什么安装失败。
- **幽灵 API**：模型编造了一个看起来合理但不存在的函数签名，你的代码写完后才发现根本无法运行。
- **版本穿越**：模型把某个框架 v3 的 API 和 v5 的 API 混在一起回答，导致不兼容的代码。
- **论文幻影**：模型编造了一篇完美的论文引用，你在 Google Scholar 上搜不到。

该 Prompt 通过强制性的**置信度标注 + 虚构禁令 + 版本敏感声明**三重机制，让模型在引用事实时从"斩钉截铁地编造"回归到"诚实地表达不确定性"。

## 🌟 核心功能

- **置信度标注**：对关键事实性引用进行内部评估，不确定的信息必须标注`[待验证]`，并建议验证路径。
- **虚构禁令**：绝对禁止编造论文标题、API 函数名、项目地址等。不确定时必须如实声明，宁可留白不捏造。
- **版本敏感**：主动声明知识对应的大致时间范围或版本区间，提醒用户确认最新版本是否存在变更。

## 💡 适用场景

- **技术选型与依赖引入**：当模型推荐工具、库或框架时，避免引入不存在的"幽灵依赖"。
- **API 调用与代码编写**：确保模型给出的函数签名、参数名称等具有真实性。
- **学术研究与文献引用**：防止模型编造论文引用，浪费检索时间。
